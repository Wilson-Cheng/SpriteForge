# Security Policy

## Scope

SpriteForge is a client-side web application. No user data is transmitted, stored on a server, or processed remotely. All file operations happen locally in the browser.

## Reporting a vulnerability

If you discover a security issue, please open a GitHub issue labelled `security` rather than a public discussion thread if the issue is sensitive. For non-sensitive issues, a normal issue is fine.

## What to report

- Cross-site scripting (XSS) via SVG or crafted project files
- Inadvertent data exfiltration
- Dependency vulnerabilities

## Response

The maintainer will respond within 7 days.
